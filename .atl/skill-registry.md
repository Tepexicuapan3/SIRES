## Project Conventions

| Path | Purpose | When to read |
| --- | --- | --- |
| `AGENTS.md` | Root operating guide for the whole repository | Start of any work; defines overall architecture and delivery flow |
| `backend/AGENTS.md` | Backend-specific rules (Django/DRF, domain-first, DB per domain) | When implementing backend features, APIs, or database changes |
| `frontend/AGENTS.md` | Frontend-specific rules (React 19, domain-first, state management) | When implementing frontend components, hooks, or UI flows |
| `docs/AGENTS.md` | Documentation rules and standards | When writing or updating documentation |
| `docs/architecture/` | Architecture decision records, context maps, dependency rules | When making architectural decisions, designing new domains, or cross-domain changes |
| `docs/guides/` | Operational guides (PR merge governance, domain DoR/DoD, incremental migration) | When planning delivery, defining done criteria, or migrating domains |
| `docs/api/` | API contracts and standards | When designing or modifying API endpoints |

## Notes

- **SDD skills** (`sdd-*`) are system-level skills for Spec-Driven Development and are managed by the orchestrator. They are not listed here.
- **Engram persistence** is enabled for this project. Use `project: SIRES_SHARED` for team‑wide memory and `project: SIRES_LOCAL` for personal notes.
